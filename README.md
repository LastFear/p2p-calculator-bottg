# Счетовод: Telegram P2P-калькулятор

`Счетовод` - это Telegram-бот с мини-приложением для учета P2P-сделок, банков, финансовых операций и личной статистики. Бот открывает Web App внутри Telegram, а данные каждого пользователя хранятся отдельно в Telegram CloudStorage.

## Что умеет бот

- Считать покупки и продажи USDT.
- Показывать прибыль, оборот, баланс USDT и баланс рублей.
- Учитывать остатки на банках.
- Вести историю сделок с фильтрами и сортировкой.
- Показывать статистику за все время, год, месяц, неделю и сегодня.
- Учитывать расходы по картам во вкладке `Финансы`.
- Делать переводы между банками и зачисления из кэшбэка.
- Показывать активность карт и последние операции.
- Строить графики по обороту и прибыли.
- Делать резервные копии и проверять целостность данных.

## Основная логика

### Покупка USDT

Покупка используется, когда ты тратишь рубли с банка и получаешь USDT.

Что происходит:

- сумма рублей списывается с выбранного банка;
- количество USDT прибавляется к балансу USDT;
- сделка попадает в историю;
- курс считается автоматически как `рубли / USDT`;
- операция участвует в обороте и статистике.

### Продажа USDT

Продажа используется, когда ты отдаешь USDT и получаешь рубли на банк.

Что происходит:

- USDT уменьшается;
- рубли зачисляются на выбранный банк;
- сделка попадает в историю;
- операция участвует в обороте и расчете прибыли.

### Бонусы

Бонусы нужны для пополнения USDT без рублевой части.

Что происходит:

- USDT прибавляется к балансу;
- рубли и банк не указываются;
- операция сохраняется в истории как бонус.

## Главная вкладка `Сделки`

На главной странице находятся:

- фильтр периода: `Все время`, `Год`, `Месяц`, `Неделя`, `Сегодня`;
- верхние карточки статистики;
- переключатель типа операции: `Покупка`, `Продажа`, `Бонусы`;
- поля ввода суммы рублей, USDT и банка;
- кнопка `Период` для ручного выбора даты и времени сделки;
- кнопка добавления сделки.

Периоды считаются не по календарю, а скользящим окном:

- `Сегодня` - текущий день;
- `Неделя` - последние 7 дней;
- `Месяц` - последние 30 дней;
- `Год` - последние 365 дней;
- `Все время` - весь архив.

Если не открывать меню `Период`, сделка добавляется с текущей датой и временем. Если дату и время выбрать вручную, сделка сохранится именно с ними.

## Верхние показатели

### Прибыль

Карточка показывает расчетную прибыль за выбранный период. Это информационный показатель: прибыль не прибавляется к рублевому балансу сама по себе.

При нажатии открывается график прибыли по месяцам. Минусовая прибыль в графике не отображается, чтобы удобнее сравнивать положительные месяцы.

### Оборот

Карточка показывает оборот за выбранный период.

При нажатии открывается график оборота отдельно по покупкам и продажам.

### Баланс USDT

Показывает текущий расчетный остаток USDT по всем сделкам и бонусам.

### Баланс рублей

Показывает текущую сумму остатков по всем банкам. Этот баланс строится из банков, а не из карточки прибыли.

## Вкладка `История`

Во вкладке истории отображаются сделки:

- покупки;
- продажи;
- бонусы.

Доступны фильтры:

- все операции;
- только покупки;
- только продажи;
- только бонусы;
- фильтр по банку;
- сортировка: новые сверху или старые сверху.

История подгружается постепенно, чтобы приложение не тормозило при большом количестве операций.

В истории можно:

- редактировать сделку;
- удалить сделку;
- восстановить удаленную операцию через кнопку отката;
- менять дату и время сделки при редактировании.

## Вкладка `Банки`

Во вкладке банков можно:

- добавить банк;
- изменить остаток на банке;
- удалить банк;
- менять порядок банков;
- видеть общий баланс по банкам;
- смотреть сделки за выбранный период по банкам;
- открыть меню резервных копий.

Остаток на банке меняется автоматически:

- при покупке USDT сумма рублей списывается;
- при продаже USDT сумма рублей зачисляется;
- при покупке картой сумма списывается;
- при переводе между банками сумма переносится;
- при переводе из `Кэшбэк / пополнение наличных` или `Зарплата` сумма зачисляется на выбранный банк или в наличные без списания с другого банка;
- при переводе в `Наличные` сумма списывается с выбранного банка и прибавляется к остатку наличных;
- при переводе из `Наличные` сумма списывается из наличных и зачисляется на выбранный банк.

Внизу вкладки есть отдельная карточка `Наличные`. Это простой остаток без статистики продаж, перегрева и сделок. Он нужен, чтобы снятые или внесенные наличные не терялись в общей картине финансов.

## Вкладка `Финансы`

Финансы нужны для учета обычных карточных расходов и переводов между своими банками.

### Покупка картой

Используется, когда ты оплатил что-то банковской картой.

Можно указать:

- банк;
- категорию траты;
- сумму;
- заметку.

Сумма списывается с выбранного банка и попадает в финансовую историю.

Основные категории трат:

- продукты;
- кафе и рестораны;
- транспорт;
- такси;
- связь и интернет;
- маркетплейсы;
- одежда;
- дом;
- здоровье;
- развлечения;
- подписки;
- долг;
- прочее.

Если выбрать категорию `Долг`, появляется переключатель:

- `Дал долг` - деньги списываются с банка, а наверху последних операций появляется плашка `Мне должны`;
- `Взял долг` - деньги прибавляются на банк, а наверху последних операций появляется плашка `Я должен`.

На долговой плашке есть кнопка уменьшения остатка. Если долг возвращают частями или ты отдаешь долг частями, можно ввести сумму погашения, и приложение оставит только остаток. Корзина на долговой плашке нужна не для погашения, а для удаления ошибочной записи с откатом баланса.

### Перевод

Перевод используется для движения денег между своими банками.

Можно выбрать:

- откуда списать;
- куда зачислить;
- сумму;
- заметку.

В поле `Откуда списать` есть отдельные пункты:

- `Кэшбэк / пополнение наличных` - внешний приход без списания с банка;
- `Зарплата` - внешний приход без списания с банка;
- `Наличные` - списание из остатка наличных;
- свои банки.

В поле `Куда зачислить` есть пункт `Наличные`. Он нужен для сценария, когда деньги сняты с банка в наличные: с выбранного банка сумма списывается, а остаток наличных увеличивается.

Примеры:

- зарплата на карту: `Зарплата -> нужный банк`;
- кэшбэк на карту: `Кэшбэк / пополнение наличных -> нужный банк`;
- снял наличные: `банк -> Наличные`;
- внес наличные на карту: `Наличные -> банк`;
- получил наличные извне: `Кэшбэк / пополнение наличных -> Наличные`.

### Карточки операций и трат

Карточки `Операций за период` и `Траты за период` открывают меню фильтров финансов.

В меню можно выбрать:

- период;
- банк;
- сортировку.

Фильтр периода влияет и на карточки, и на список операций. Фильтр банка и сортировка применяются к списку в меню.

### Активность карт

Блок активности показывает по каждому банку:

- последние операции;
- сколько времени карта была без покупок;
- краткую историю последних расходов.

## Резервные копии

Меню резервных копий находится во вкладке `Банки` внизу.

Возможности:

- `Экспорт` - сформировать полный JSON-бэкап;
- `Скопировать` - скопировать бэкап;
- `Применить` - восстановить данные из вставленного JSON;
- `Очистить` - очистить поле бэкапа;
- `Создать бэкап` - вручную создать резервную копию;
- `Загрузить последний бэкап` - подставить последний автоматический бэкап в поле;
- `Проверить данные` - запустить проверку целостности.

Автоматические бэкапы:

- создаются в 00:00, если данные изменились;
- если приложение было закрыто в 00:00, пропущенный бэкап создается при следующем открытии приложения, но только если данные отличаются от последней копии;
- хранятся в Telegram CloudStorage;
- хранится 1 последняя автоматическая копия;
- перед применением импортированного бэкапа создается страховочная копия.

## Проверка данных

Проверка данных помогает найти потенциальные ошибки:

- сделки без банка;
- неизвестные типы операций;
- суммы меньше или равные нулю;
- операции из будущего;
- банки, которых нет в списке;
- похожие дубли;
- отрицательные остатки банков.

Проверка сделок и финансовых операций выполняется только начиная с `25.06.2026 00:00`. Старые операции до этой даты не учитываются в отчете, чтобы не мешали старые записи, созданные до появления части новых правил.

## Где хранятся данные

Данные хранятся в Telegram CloudStorage отдельно для каждого Telegram-пользователя.

Это значит:

- у каждого пользователя свои сделки, банки и финансы;
- пользователи не видят данные друг друга;
- сервер Render не является основной базой данных;
- если пользователь откроет бота со своего Telegram-аккаунта, он увидит свои данные.

В CloudStorage сохраняются:

- сделки;
- банки;
- покупки картой;
- переводы;
- автоматические бэкапы;
- служебная информация о бэкапах.

## Файлы проекта

- `index.html` - мини-приложение Telegram Web App.
- `bot.py` - Flask-бот, который отвечает Telegram и отдает ссылку на мини-приложение.
- `app.py` - совместимость для запуска через WSGI.
- `render.yaml` - конфигурация Render.

## Переменные окружения

Для запуска нужны:

```text
BOT_TOKEN=токен_бота_из_BotFather
WEBAPP_URL=https://адрес-приложения/
```

На Render эти переменные задаются в настройках сервиса.

## Локальный запуск

Пример для PowerShell:

```powershell
$env:BOT_TOKEN="токен_бота"
$env:WEBAPP_URL="https://example.com/"
python bot.py
```

Для реального использования без запуска на ПК проект разворачивается на Render. Подробности есть в `DEPLOY.md`.

## Важные ограничения

- Это личный учетный инструмент, а не банковская система.
- Если вручную поменять остаток банка, статистика будет считать от нового остатка.
- Если удалить сделку или финансовую операцию, приложение откатывает связанные изменения балансов.
- При восстановлении из бэкапа текущие данные заменяются данными из бэкапа.
- Telegram CloudStorage удобен для текущего масштаба, но при очень большом архиве может понадобиться отдельная база данных.

## Рекомендуемый порядок использования

1. Добавить банки во вкладке `Банки`.
2. Указать стартовые остатки по банкам.
3. Добавлять покупки и продажи USDT через вкладку `Сделки`.
4. Обычные расходы вести во вкладке `Финансы`.
5. Переводы между картами вести через `Финансы -> Перевод`.
6. Кэшбэк, зарплату или внешний приход зачислять через перевод на нужный банк или в наличные.
7. Снятие наличных учитывать через перевод `банк -> Наличные`, внесение наличных - через `Наличные -> банк`.
8. Периодически открывать резервные копии и нажимать `Проверить данные`.

## Коротко

`Счетовод` помогает видеть реальную картину: сколько USDT есть, сколько рублей лежит на банках и в наличных, какой оборот прошел, какая прибыль получилась и куда уходят деньги с карт.
